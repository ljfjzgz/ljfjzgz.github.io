# 关于是否计费规则整理

## 一、基本概念

**是否计费规则**：用于判断费用项是否应该被计算的规则体系，是计费引擎中费用项配置的重要组成部分。

**核心作用**：
- 控制费用项的计费开关
- 基于业务单据参数进行条件判断
- 实现灵活的计费控制逻辑

## 二、配置结构

标准配置结构：

```json
"是否计费": {
  "默认计费": "[是/否]",
  "判断规则": {
    "因子值": "[因子编码]",
    "操作符": "[操作符]",
    "匹配值": "[匹配值]"
  }
}
```

**字段说明**：
| 字段 | 说明 |
|------|------|
| 默认计费 | 费用项的默认计费状态 |
| 因子值 | 业务单据中的因子编码 |
| 操作符 | 操作符 |
| 匹配值 | 用于匹配的值 |

## 三、因子的定义与使用

**重要说明**：
- 因子是计费引擎中用于计算费用的基础元素，业务单据会注册到计费引擎并创建对应的因子
- 因子在引擎中的唯一标识 = 所属单据编码 + 因子编码
- **因子编码只能传基于模型关联的业务单据的因子编码**，即只能使用与当前模型关联的业务单据所对应的因子
- 上游系统传递的业务单据字段必须与引擎中注册的因子对应
- 因子的类型决定了可以使用的操作符

## 四、操作符

| 操作符 | 说明 | 适用类型 |
|--------|------|----------|
| $Equal$ | 等于 | 所有类型 |
| $NotEqual$ | 不等于 | 所有类型 |
| $GreaterThan$ | 大于 | 数值 |
| $LessThan$ | 小于 | 数值 |
| $GreaterOrEqual$ | 大于等于 | 数值 |
| $LessOrEqual$ | 小于等于 | 数值 |
| $In$ | 包含于 | 数组 |
| $NotIn$ | 不包含于 | 数组 |
| $AnyContain$ | 任一包含 | 数组 |
| $AllContain$ | 全部包含 | 数组 |
| $StartsWith$ | 开头是 | 字符串 |
| $EndsWith$ | 结尾是 | 字符串 |
