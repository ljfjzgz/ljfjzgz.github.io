# 计费引擎模型计费规则抽象

## 计费要素

计费引擎的计费规则以三类核心要素为基础，结合价格信息、计算规则等其他配置，共同构成完整的费用计算体系：

### 1. 因子（Factor）
- **定义**：计费计算的基础数据来源，绑定到业务单据的具体字段
- **类型**：数值、字符串、数组
- **用途**：
  - 区间判断（如重量、体积用于阶梯计费）
  - 费用计算（如重量、件数作为计费基础）
  - 条件判断（如目的国、商品类型用于条件计费）
- **引用方式**：通过因子编码引用，因子编码在所属单据内唯一

### 2. 通用计费要素
- **定义**：用于费用计算和条件判断的操作符集合
- **类型**：
  - 基础操作要素：NONE（无操作）
  - 算术运算符：ADD, SUB, MUL, DIV
  - 逻辑运算符：AND, OR, NOT
  - 操作符：$Equal$, $NotEqual$, $GreaterThan$, $LessThan$, $GreaterOrEqual$, $LessOrEqual$, $AnyContain$, $AllContain$
- **用途**：
  - 基础操作：直接使用原始值，适用于单因子场景、跳过区间判断、跳过条件检查等
  - 复杂数学计算（如体积计算、费用汇总）
  - 多条件组合判断（如目的国+重量+商品类型）
  - 数组元素判断（如商品名称包含特定关键词）

### 3. 费用项（Fee Item）
- **定义**：计费规则中的具体费用项目
- **引用能力**：可以引用同一模型内的其他费用项
- **用途**：
  - 费用汇总（如总运费=基础运费+燃油附加费）
  - 比例计算（如管理费=总运费×10%）
  - 多级费用结构（如基础费用+附加费用）

### 要素使用原则
- 所有因子必须在计费引擎中已维护，通过因子编码引用
- 通用计费要素用于构建复杂的计算逻辑和条件判断
- 费用项用于构建多级费用结构和相互依赖的费用计算
- 三类要素相互配合，构成灵活而强大的计费规则体系

## 计费规则

### 阶梯计费 - 分段计费

计费规则：阶梯计费-分段计费
规则明细：
{
  区间因子：[区间因子],
  区间：(开始值, 结束值],
  计费因子：[计费因子],
  区间价格：[金额],
  最小值：[最小值],
  最大值：[最大值]
}
{
  区间因子：[区间因子],
  区间：(开始值, 结束值],
  计费因子：[计费因子],
  区间价格：[金额],
  最小值：[最小值],
  最大值：[最大值]
}

**校验规则**：
- 区间因子和计费因子必须是计费引擎中已维护的因子编码。
- 同一个费用项下，区间因子必须是相同的。
- 且区间必须是衔接的，区间按左开右闭方式定义，例如（0, 10]、(10, 20]、(20, 30] 等。
- 区间价格、最小值、最大值必须是数值类型，保留 6 位小数。
- 最大值的金额必须大于最小值的金额

**计算逻辑**：
- 根据「区间因子值」在规则明细中找到匹配的区间
- 取匹配区间的「计费因子值」×「区间价格」
- 上下限处理
    - 若有最小值：计算结果与最小值比较，取较大值
    - 若有最大值：计算结果与最大值比较，取较小值
    - 若同时有最小值和最大值：先与最小值比较，再与最大值比较

**示例**：包裹数量阶梯计费（核重计费）

```json
[
  {
    "区间因子": "PackageCount",
    "区间": "(0, 600]",
    "计费因子": "CalculatedWeight",
    "区间价格": "0",
    "最小值": "250",
    "最大值": "10000"
  },
  {
    "区间因子": "PackageCount",
    "区间": "(600, 1500]",
    "计费因子": "CalculatedWeight",
    "区间价格": "0.65",
    "最小值": "0",
    "最大值": "10000"
  },
  {
    "区间因子": "PackageCount",
    "区间": "(1500, 100000]",
    "计费因子": "CalculatedWeight",
    "区间价格": "0.6",
    "最小值": "0",
    "最大值": "10000"
  }
]
```

**示例说明**：
- 区间因子：PackageCount（包裹数量）
- 计费因子：CalculatedWeight（核重）
- 价格规则：
  - 包裹数量 ≤ 600：低消250元
  - 包裹数量 600 ＜ x ≤ 1500：0.65元/kg
  - 包裹数量 > 1500：0.6元/kg

**示例计算**：
- 包裹数量500，核重100kg：
  - 计算费用：100kg × 0元/kg = 0元
  - 应用最小值：取250元
  - 最终费用：250元

- 包裹数量1000，核重200kg：
  - 计算费用：200kg × 0.65元/kg = 130元
  - 应用上下限：130元在0-10000之间
  - 最终费用：130元

- 包裹数量2000，核重300kg：
  - 计算费用：300kg × 0.6元/kg = 180元
  - 应用上下限：180元在0-10000之间
  - 最终费用：180元


### 阶梯计费 - 累进计费

计费规则：阶梯计费-累进计费
规则明细：
{
  区间因子：[区间因子],
  区间：[开始值, 结束值),
  区间价格：[金额],
  最小值：[最小值],
  最大值：[最大值]
}
{
  区间因子：[区间因子],
  区间：[开始值, 结束值),
  区间价格：[金额],
  最小值：[最小值],
  最大值：[最大值]
}

**校验规则**：
- 区间因子必须是计费引擎中已维护的因子编码，且必须是数值类型。
- 同一个费用项下，区间因子必须是相同的。
- 区间必须是衔接的，区间按左闭右开方式定义，例如：[0, 10)、[10, 20)、[20, 30) 等。
- 区间价格、最小值、最大值必须是数值类型，保留 6 位小数。
- 最大值的金额必须大于最小值的金额

**计算逻辑**：
- 遍历所有区间，从第一个区间开始计算
- 对于每个区间：
  - 计算该区间的计费数量 = min(当前重量, 区间结束值) - max(0, 区间开始值)
  - 该区间费用 = 计费数量 × 区间价格
- 累加所有区间的费用得到最终结果
- 上下限处理
  - 若有最小值：计算结果与最小值比较，取较大值
  - 若有最大值：计算结果与最大值比较，取较小值
  - 若同时有最小值和最大值：先与最小值比较，再与最大值比较

**示例**：重量累进计费（含上下限）

```json
[
  {
    "区间因子": "Weight",
    "区间": "[0, 10)",
    "区间价格": "10",
    "最小值": "50",
    "最大值": "200"
  },
  {
    "区间因子": "Weight",
    "区间": "[10, 50)",
    "区间价格": "8",
    "最小值": "80",
    "最大值": "500"
  },
  {
    "区间因子": "Weight",
    "区间": "[50, 10000)",
    "区间价格": "6",
    "最小值": "200",
    "最大值": "800"
  }
]
```

**示例说明**：
- 区间因子：Weight（重量）
- 价格规则（累进计费）：
  - 0 ≤ Weight < 10kg：10元/kg，最低50元，最高200元
  - 10 ≤ Weight < 50kg：8元/kg，最低80元，最高500元
  - 50 ≤ Weight < 10000kg：6元/kg，最低200元，最高800元

**示例计算**：
- 重量为2kg（触发最小值）：
  - 0-2kg区间：2kg × 10元/kg = 20元
  - 应用最小值：取50元
  - 最终费用：50元

- 重量为15kg（正常情况）：
  - 0-10kg区间：10kg × 10元/kg = 100元
  - 10-15kg区间：5kg × 8元/kg = 40元
  - 计算总费用：100 + 40 = 140元
  - 应用上下限：140元在80-500之间
  - 最终费用：140元

- 重量为60kg（正常情况）：
  - 0-10kg区间：10kg × 10元/kg = 100元
  - 10-50kg区间：40kg × 8元/kg = 320元
  - 50-60kg区间：10kg × 6元/kg = 60元
  - 计算总费用：100 + 320 + 60 = 480元
  - 应用上下限：480元在200-800之间
  - 最终费用：480元


### 按比例计费

计费规则：比例型计费
规则明细：
{
  "比例": "[比例]",
  "计算因子": {
    "运算符": "[运算符]",
    "操作数": [ 
      {
        "运算符": "[运算符]",
        "操作数": ["[操作数因子]", "[操作数因子]"]
      },
      {
        "运算符": "[运算符]",
        "操作数": ["[操作数因子]", "[操作数因子]"]
      }
    ]
  },
  "最小值": "[最小值]",
  "最大值": "[最大值]"
}

**说明**：
- 操作数可以是因子编码（如 "Weight"）或嵌套的计算因子结构
- 通过嵌套计算因子，可以支持复杂的运算表达式，如 (a + b) - (c - d)
- 运算符支持 NONE（无运算）、ADD（加）、SUB（减）、MUL（乘）、DIV（除）

**校验规则**：
- 比例验证：比例必须是百分比样式，如12%、5%等，比例数值必须在0-100%之间，数字精度建议6位数
- 计算因子验证：运算符必须是系统支持的运算符（ADD、SUB、MUL、DIV、NONE），操作数必须是计费引擎中已维护的因子编码或常量数值，操作数可以是一个或多个
- 上下限验证：最小值和最大值必须是数值类型，保留 6 位小数，最大值必须大于最小值

**计算逻辑**：
1. 计算计费因子值：如果计算因子包含运算符，按运算符对操作数进行运算；如果计算因子只包含单个因子编码，直接获取该因子的值
2. 计算费用：公式=计费因子计算结果×比例
3. 上下限处理：若有最小值，计算结果与最小值比较，取较大值；若有最大值，计算结果与最大值比较，取较小值；若同时有最小值和最大值，先与最小值比较，再与最大值比较

**示例**：复杂运算表达式 (a + b * c) - (d - e / f)

```json
{
  "比例": "5%",
  "计算因子": {
    "运算符": "SUB",
    "操作数": [
      {
        "运算符": "ADD",
        "操作数": ["a", {
          "运算符": "MUL",
          "操作数": ["b", "c"]
        }]
      },
      {
        "运算符": "SUB",
        "操作数": ["d", {
          "运算符": "DIV",
          "操作数": ["e", "f"]
        }]
      }
    ]
  },
  "最小值": "0",
  "最大值": "1000"
}
```

**示例说明**：
- 该示例计算 (a + b * c) - (d - e / f) 的结果，然后乘以 5% 的比例
- 第一层运算：b * c
- 第二层运算：a + (b * c)
- 第三层运算：e / f
- 第四层运算：d - (e / f)
- 第五层运算：(a + b * c) - (d - e / f)
- 最后应用 5% 的比例计算最终费用

**示例2**：简单场景（单因子）

```json
{
  "比例": "8%",
  "计算因子": {
    "运算符": "NONE",
    "操作数": ["CalculatedWeight"]
  },
  "最小值": "0",
  "最大值": "5000"
}
```

**示例2说明**：
- 该示例直接获取核重值，然后乘以 8% 的比例
- 核重 100kg：费用 = 100 × 0.08 = 8 元


### 条件型计费

计费规则：条件型计费
规则明细：
{
  "条件规则": {
    "逻辑运算符": "[逻辑运算符]",
    "条件组合": [
      {
        "逻辑运算符": "[逻辑运算符]",
        "条件组合": [
          {
            "条件": {
              "计算因子": {
                "运算符": "[运算符]",
                "操作数": ["[操作数因子]"]
              },
              "匹配规则": "[操作符]",
              "匹配值": "[匹配值]"
            }
          },
          {
            "条件": {
              "计算因子": {
                "运算符": "[运算符]",
                "操作数": [
                  "[操作数因子]",
                  {
                    "运算符": "[运算符]",
                    "操作数": ["[操作数因子]", "[操作数因子]"]
                  }
                ]
              },
              "匹配规则": "[操作符]",
              "匹配值": "[匹配值]"
            }
          }
        ]
      },
      {
        "逻辑运算符": "[逻辑运算符]",
        "条件组合": [
          {
            "条件": {
              "计算因子": {
                "运算符": "[运算符]",
                "操作数": ["[操作数因子]"]
              },
              "匹配规则": "[操作符]",
              "匹配值": "[匹配值]"
            }
          }
        ]
      }

  },
  "命中规则": {

        },
        "最小值": "[最小值]",
        "最大值": "[最大值]"
      }
    ]
  },
  "未命中规则": {
    "计费规则": "阶梯计费-分段计费",
    "规则明细": [
      {
        "区间因子": "[区间因子]",
        "区间": "(开始值, 结束值]",
        "计费因子": "[计费因子]",
        "区间价格": "[金额]",
        "最小值": "[最小值]",
        "最大值": "[最大值]"
      },
      {
        "区间因子": "[区间因子]",
        "区间": "(开始值, 结束值]",
        "计费因子": "[计费因子]",
        "区间价格": "[金额]",
        "最小值": "[最小值]",
        "最大值": "[最大值]"
      }
    ]
  }
}

**校验规则**：
- 条件规则校验：
  - 必须包含"逻辑运算符"和"条件组合"两个字段
  - 逻辑运算符必须是AND、OR之一
  - 条件组合必须是数组形式，且至少包含一个条件
  - 每个条件必须包含计算因子、匹配规则和匹配值
- 计算因子校验：
  - 运算符必须是有效的算术运算符（ADD、SUB、MUL、DIV、NONE）
  - 操作数可以是因子编码、常量数值或嵌套的计算因子结构
  - 操作数中的因子编码必须是计费引擎中已维护的因子
- 匹配规则校验：
  - 匹配规则必须是有效的操作符（$Equal$、$NotEqual$、$GreaterThan$、$LessThan$、$GreaterOrEqual$、$LessOrEqual$、$AnyContain$、$AllContain$）
  - 匹配值必须与计算因子的返回类型相匹配
- 命中规则和未命中规则校验：
  - 必须包含"计费规则"和"规则明细"两个字段
  - 计费规则必须是系统支持的规则类型（如"比例型计费"、"阶梯计费-分段计费"等）
  - 规则明细必须符合对应计费规则的结构要求
- 其他校验：
  - 条件组合支持嵌套，嵌套层级不建议超过3层
  - 命中规则和未命中规则可以分别使用不同类型的计费规则

**计算逻辑**：
1. 条件计算：
   - 遍历所有条件，根据条件中的"计算因子"计算因子的值
   - 若计算因子包含运算符（如ADD、SUB、MUL、DIV），则对操作数进行相应运算
   - 若计算因子只包含单个因子编码，则直接获取该因子的值
   - 将计算结果与"匹配值"进行对比，根据"匹配规则"判断条件是否满足

2. 条件规则计算：
   - 根据"条件规则"中的"逻辑运算符"和"条件组合"递归计算最终结果
   - AND：所有条件都满足才返回true
   - OR：任意一个条件满足就返回true
   - NOT：对条件结果取反

3. 规则执行：
   - 若条件规则计算结果为true，则执行"命中规则"
   - 若条件规则计算结果为false，则执行"未命中规则"
   - 根据规则中的"计费规则"类型，调用对应的计费逻辑进行费用计算

**未命中规则说明**：
- 未命中规则为可选配置
- 如果未配置未命中规则，条件不满足时返回null
- 如果配置了未命中规则，条件不满足时执行未命中规则中的计费逻辑

**运算符说明**

| 运算符 | 含义 | 适用类型 | 示例 |
|--------|------|----------|------|
| NONE | 无运算 | 所有类型 | 当计算因子仅引用单个因子编码或常量，无需运算时使用。操作数仅包含一个元素。例如：`NONE(Length)` 直接获取Length因子值，`NONE(105)` 直接获取常量105 |
| ADD | 加法 | 数值类型 | 例如：`ADD(Length, Width, Height)` 计算长宽高之和 |
| SUB | 减法 | 数值类型 | 例如：`SUB(TotalWeight, TareWeight)` 计算净重 |
| MUL | 乘法 | 数值类型 | 例如：`MUL(Length, Width, Height)` 计算体积 |
| DIV | 除法 | 数值类型 | 例如：`DIV(Length * Width * Height, 6000)` 计算体积重量 |

**操作符说明**

| 操作符 | 含义 | 适用类型 | 示例 |
|--------|------|----------|------|
| $AnyContain$ | 数组中任意一个元素包含指定值 | 数组类型 | 例如：`SkuDeclareNameCnArr $AnyContain$ "空调"` 表示商品名称数组中只要有一个名称包含"空调"，就满足条件 |
| $AllContain$ | 数组中所有元素都包含指定值 | 数组类型 | 例如：`SkuDeclareNameCnArr $AllContain$ "电器"` 表示商品名称数组中所有名称都必须包含"电器"，才满足条件 |
| $Equal$ | 完全等于指定值 | 所有类型 | 例如：`CustomerLevel $Equal$ "VIP"` 表示客户等级必须完全等于"VIP"，才满足条件 |
| $NotEqual$ | 不等于指定值 | 所有类型 | 例如：`CustomerLevel $NotEqual$ "普通"` 表示客户等级不等于"普通"，就满足条件 |
| $GreaterThan$ | 大于指定值 | 数值类型 | 例如：`FeeWeight $GreaterThan$ 20` 表示计费重量大于20kg，就满足条件 |
| $LessThan$ | 小于指定值 | 数值类型 | 例如：`FeeWeight $LessThan$ 10` 表示计费重量小于10kg，就满足条件 |
| $GreaterOrEqual$ | 大于等于指定值 | 数值类型 | 例如：`FeeWeight $GreaterOrEqual$ 22` 表示计费重量大于等于22kg，就满足条件 |
| $LessOrEqual$ | 小于等于指定值 | 数值类型 | 例如：`FeeWeight $LessOrEqual$ 50` 表示计费重量小于等于50kg，就满足条件 |

**支持的逻辑运算符**

| 逻辑运算符 | 含义 | 示例 |
|-----------|------|------|
| **AND** | 逻辑与，所有条件都必须满足 | 条件1 AND 条件2，表示两个条件都要满足 |
| **OR** | 逻辑或，任意一个条件满足即可 | 条件1 OR 条件2，表示只要一个条件满足 |
| **NOT** | 逻辑非，条件不满足 | NOT 条件1，表示条件1不满足时才满足 |

### 实报实销

计费规则：实报实销
规则明细：
    {
      实报实销
    }

**规则说明**：
- 实报实销无需维护报价，直接使用业务系统传递的费用数据
- 引擎仅做数据传递和格式处理，不进行费用计算
- 适用于需要直接使用实际发生费用的场景

**校验规则**：
- 业务系统传参验证：业务系统传递的实报实销费用数据必须包含费用项、金额和币种字段
- 数据完整性验证：每个费用项必须同时包含金额和币种信息

**计算逻辑**：
1. 接收业务系统传递的实报实销费用数据，包含费用项、金额和币种（每个金额必须携带币种）
2. 对每个费用项进行参数校验，确保费用项、金额和币种字段完整
3. 直接使用业务系统传递的费用项、金额和币种，不做任何计算处理
4. 若同一费用项有多个金额，输出对应数量的同名费用项金额（每个金额需单独处理，且每个金额都必须包含币种）
