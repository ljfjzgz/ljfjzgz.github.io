# 因子配置规则

## 一、背景

随着计费引擎的发展，因子作为计费规则的核心组成部分，需要更加标准化和集中化管理。为了支持多业务系统的复杂计费需求，确保因子与业务单据的正确绑定，提高因子配置的效率和准确性，特制定本因子配置规则方案。

## 二、术语定义

| 术语 | 定义 |
|------|------|
| 因子 | 计费引擎中用于计算费用的基础元素，包括用于区间判断、费用计算、条件判断等的各类元素 |
| 所属单据 | 因子关联的业务单据类型，如TruckOrder、Waybill等 |
| 因子编码 | 从所属单据内选择的字段编码，是因子在所属单据内的唯一标识，用于计费规则中引用 |
| 业务字段 | 因子绑定的业务系统字段，根据因子编码自动关联 |
| 字段来源 | 字段所属的单据层级，枚举值：主单据、子单据，根据业务字段元数据自动带出 |
| 数据类型 | 因子的值类型，如数值、字符串、数组 |

## 三、功能模块

### 3.1 因子配置管理

#### 3.1.1 唯一性判断

**引擎中的唯一性定义**：
- 因子在引擎中的唯一标识 = `所属单据编码` + `因子编码`
- 示例：`TruckOrder_WEIGHT` 表示 TruckOrder 单据下的 WEIGHT 因子

**唯一性规则**：
- 同一所属单据内的因子编码必须唯一
- 不同所属单据的因子编码可以重复

**校验时机**：
- 新增因子时：检查因子编码在所选所属单据内是否已存在
- 编辑因子时：检查因子编码是否与同一所属单据内的其他因子重复
- 导入因子时：检查导入数据中的因子编码在对应所属单据内是否已存在

**校验逻辑**：
1. 系统根据所属单据和因子编码查询数据库
2. 如果存在相同所属单据和因子编码的记录，则提示"因子编码在该单据内已存在"
3. 校验通过后，允许保存或导入

**错误提示**：因子编码在所属单据内已存在，请修改因子编码或选择其他所属单据

| 因子编码 | 文本输入框 | 支持模糊查询 |
| 因子名称 | 文本输入框 | 支持模糊查询 |
| 数据类型 | 下拉选择框 | 可选：数值、字符串、数组 |
| 启用状态 | 下拉选择框 | 可选：全部、启用、禁用 |

**列表字段**：
| 字段名称 | 说明 | 操作 |
|----------|------|------|
| 序号 | 自动生成的序号 | - |
| 因子ID | 系统生成的唯一标识 | - |
| 因子编码 | 因子在所属单据内的唯一编码 | - |
| 因子名称 | 因子的名称，便于识别 | - |
| 所属单据 | 因子关联的业务单据类型 | - |
| 业务字段 | 绑定的业务系统字段 | - |

| 数据类型 | 因子的值类型 | - |
| 启用状态 | 显示启用/禁用状态 | - |

**页面按钮**：新增、导入、导出、刷新

**列表操作**：点击编辑进入编辑页面，点击启用/禁用切换因子状态

#### 3.1.3 新增页面

**表单字段**：
| 字段名称 | 控件类型 | 是否必填 | 说明 | 校验规则 |
|----------|----------|----------|------|----------|
| 所属单据 | 下拉选择框 | 是 | 选择因子关联的业务单据类型 | 必选 |
| 因子编码 | 下拉选择框 | 是 | 选择所属单据内的字段编码 | 必选 |
| 因子名称 | 文本输入框 | 是 | 因子的名称 | 非空，长度≤50 |
| 业务字段 | 只读文本 | 是（自动带出） | 绑定的业务系统字段（根据因子编码自动带出） | 不可修改 |
| 字段来源 | 只读文本 | 是（自动带出） | 字段所属的单据层级（主单据/子单据），根据业务字段元数据自动带出 | 不可修改 |
| 数据类型 | 只读文本 | 是（自动带出） | 因子的值类型（根据因子编码自动带出） | 不可修改 |
| 描述 | 文本域 | 否 | 因子的详细描述 | 长度≤200 |
| 启用状态 | 开关 | 是 | 因子的启用状态 | 默认为启用 |

**交互逻辑**：
1. 选择所属单据后，系统自动加载该单据内的所有字段编码到因子编码下拉框
2. 选择因子编码后，系统根据字段元数据**自动带出对应的业务字段、字段来源和数据类型**
3. 业务字段、字段来源和数据类型为**只读**，不可修改，确保与业务单据字段定义一致
4. 点击保存后，系统进行数据校验，通过后保存并返回列表页面

#### 3.1.4 编辑页面

**表单字段**：
| 字段名称 | 控件类型 | 是否可修改 | 说明 |

| 因子名称 | 文本输入框 | 是 | 因子的名称 |
| 业务字段 | 只读文本 | 否 | 绑定的业务系统字段（根据因子编码自动带出） |
| 字段来源 | 只读文本 | 否 | 字段所属的单据层级（主单据/子单据），根据业务字段元数据自动带出 |
| 数据类型 | 只读文本 | 否 | 因子的值类型（根据因子编码自动带出） |
| 描述 | 文本域 | 是 | 因子的详细描述 |
| 启用状态 | 开关 | 是 | 因子的启用状态 |

**交互逻辑**：
1. 所属单据、因子编码、业务字段、数据类型均不可修改，确保因子标识的稳定性
2. 点击保存后，系统进行数据校验，通过后保存并返回列表页面

### 3.3 导入导出功能

#### 3.3.1 导入功能

**导入流程**：
1. 点击"导入"按钮，弹出导入模态框
2. 下载导入模板（Excel格式）
3. 按模板填写因子数据
4. 上传Excel文件
5. 系统验证数据格式和唯一性
6. 显示导入结果，包括成功数量、失败数量和失败原因
7. 点击完成，返回列表页面

**导入模板字段**：所属单据、因子编码、因子名称、描述、启用状态

**导入验证规则**：
- 所属单据必须是已注册的业务单据类型
- 因子编码必须是所属单据内已存在的字段编码
- 因子编码在所属单据内必须唯一（唯一性校验）
- 启用状态必须是"启用"或"禁用"

#### 3.3.2 导出功能

**导出流程**：
1. 点击"导出"按钮
2. 选择导出范围（当前页/全部）
3. 系统生成Excel文件
4. 浏览器下载导出文件

**导出字段**：与列表页面字段一致

## 四、与其他模块的关系

### 4.1 与业务单据的关系

- 因子配置前需先注册业务单据类型
- 因子绑定到具体业务单据的字段
- 不同单据的因子编码可重复，同一单据内必须唯一

### 4.2 与计费规则的关系

- **因子引用**：计费规则中使用因子进行费用计算
- **因子状态影响**：禁用的因子不能被新的计费规则引用
- **因子变更影响**：因子变更时，系统需要检查并提示对现有计费规则的影响
- **因子值验证**：
  - **阶梯计费**：区间因子必须是数值类型（整数或小数）
  - **比例型计费**：计算因子必须是数值类型，用于与比例计算最终费用
    - 计算因子可以是业务单据的字段值，也可以是模型中配置的费用项
  - **条件型计费**：条件因子根据操作符的数据类型要求：
    - `$Equal$`（等于） / `$NotEqual$`（不等于）：支持数值、字符串、数组类型
    - `$GreaterThan$`（大于） / `$LessThan$`（小于） / `$GreaterOrEqual$`（大于等于） / `$LessOrEqual$`（小于等于）：仅支持数值类型
    - `$AnyContain$`（任一包含） / `$AllContain$`（全部包含）：仅支持数组类型
- **费用项引用**：计费规则中可以直接引用同一模型内已配置的其他费用项的费用金额作为计算因子
  - 引用范围：仅限当前计费模型中已配置的费用项
  - 引用格式：费用项编码（如 FEE_SHIPPING）
  - 依赖处理：被引用的费用项必须先于当前费用项计算完成
  - 循环引用检测：系统需检测并防止循环引用（如A依赖B，B依赖A）

### 4.3 与维度配置的关系

- **维度与因子的区别**：
  - 维度：用于业务筛选和分组，侧重业务分类
  - 因子：用于费用计算，侧重数值处理
- **数据来源**：两者都可以绑定到业务单据的字段
- **协同使用**：维度用于确定计费场景，因子用于计算具体费用

## 五、边界情况处理

1. **因子重名处理**：通过所属单据隔离，允许不同单据使用相同的因子编码
2. **因子删除保护**：被引用的因子不允许删除，避免影响现有计费规则
3. **数据类型变更**：数据类型一旦设置，不允许修改，避免类型不兼容问题
4. **业务单据变更**：业务单据禁用时，相关因子自动禁用
5. **字段编码变更**：业务单据字段编码变更时，系统需要提示对相关因子的影响
