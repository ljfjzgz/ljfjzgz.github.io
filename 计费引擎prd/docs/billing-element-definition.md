# 计费要素标准定义

## 一、通用要素

### 1. 基础操作要素

#### 1.1 定义
基础操作要素是计费计算的基础操作符号，用于处理简单的操作需求。

#### 1.2 类型

| 操作符 | 含义 | 适用类型 | 说明 |
|--------|------|----------|------|
| NONE | 无操作 | 所有类型 | 直接使用原始值，无需任何运算。适用于单因子场景、跳过区间判断、跳过条件检查等通用场景 |

#### 1.3 示例

**NONE 操作符**
```json
{
  "运算符": "NONE",
  "操作数": ["Length"]
}
```
**说明**：直接获取Length因子的值，无需任何运算，适用于单因子场景。

### 1.4 应用场景与限制

**阶梯计费-分段计费/累进计费**
- **区间因子使用NONE**：
  - 含义：不需要区间判断，直接使用固定单价
  - 限制：系统应跳过区间判断逻辑，不应显示或允许配置多个区间，仅使用单一固定单价
- **计费因子使用NONE**：
  - 含义：直接使用原始因子值，不进行额外计算
  - 限制：无特殊限制，可与其他配置正常配合

**比例型计费**
- **计算因子使用NONE**：
  - 含义：直接使用原始因子值作为计算基础
  - 限制：比例值必须配置为固定值
- **比例因子使用NONE**：
  - 含义：使用固定比例，不基于其他因子动态计算
  - 限制：比例值必须为常量数值

**条件型计费**
- **条件因子使用NONE**：
  - 含义：不需要条件判断，直接应用计费规则
  - 限制：系统应跳过条件判断逻辑，不应显示或允许配置条件组合
- **计算因子使用NONE**：
  - 含义：直接使用原始因子值，不进行额外计算
  - 限制：无特殊限制，可与其他配置正常配合

**实报实销**
- 无

**通用规则与限制**
- **适用场景**：任何需要直接使用原始值而不需要额外运算的场景，任何需要跳过特定判断逻辑（如区间判断、条件检查）的场景
- **操作数限制**：NONE的操作数只能包含一个元素（单个因子或常量）
- **系统行为**：当使用NONE时，相关的计算或判断逻辑应被跳过
- **配置验证**：配置验证时应检查NONE使用的合理性，确保不会与其他配置冲突

### 2. 算术运算符

#### 2.1 定义
算术运算符用于执行数学运算，是计算费用时的核心操作符号。

#### 2.2 类型

| 运算符 | 含义 | 适用类型 | 说明 |
|--------|------|----------|------|
| ADD | 加法 | 数值类型 | 对多个操作数进行加法运算 |
| SUB | 减法 | 数值类型 | 对操作数进行减法运算 |
| MUL | 乘法 | 数值类型 | 对多个操作数进行乘法运算 |
| DIV | 除法 | 数值类型 | 对操作数进行除法运算 |

#### 2.3 示例

**ADD 运算符**
```json
{
  "运算符": "ADD",
  "操作数": ["Length", "Width", "Height"]
}
```
**说明**：计算长宽高之和，即 Length + Width + Height，适用于需要求和的场景。

**SUB 运算符**
```json
{
  "运算符": "SUB",
  "操作数": ["TotalWeight", "TareWeight"]
}
```
**说明**：计算净重，即 TotalWeight - TareWeight，适用于需要差值的场景。

**MUL 运算符**
```json
{
  "运算符": "MUL",
  "操作数": ["Length", "Width", "Height"]
}
```
**说明**：计算体积，即 Length × Width × Height，适用于需要乘积的场景。

**DIV 运算符**
```json
{
  "运算符": "DIV",
  "操作数": [
    {
      "运算符": "MUL",
      "操作数": ["Length", "Width", "Height"]
    },
    "6000"
  ]
}
```
**说明**：计算体积重量，即 (Length × Width × Height) ÷ 6000，适用于需要除法运算的场景。

#### 2.4 特性
- **通用性**：适用于所有需要数学计算的场景
- **嵌套支持**：可以嵌套使用，构建复杂的计算表达式
- **常量支持**：操作数中可以包含常量数值

### 3. 逻辑运算符

#### 3.1 定义
逻辑运算符用于条件判断，在条件型计费规则中用于组合多个条件。

#### 3.2 类型

| 逻辑运算符 | 含义 | 说明 |
|-----------|------|------|
| AND | 逻辑与 | 所有条件都必须满足才返回true |
| OR | 逻辑或 | 任意一个条件满足就返回true |
| NOT | 逻辑非 | 对条件结果取反 |

#### 3.3 示例

**AND 运算符**
```json
{
  "逻辑运算符": "AND",
  "条件组合": [
    {
      "条件": {
        "计算因子": {
          "运算符": "NONE",
          "操作数": ["DestinationCountry"]
        },
        "匹配规则": "$Equal$",
        "匹配值": "美国"
      }
    },
    {
      "条件": {
        "计算因子": {
          "运算符": "NONE",
          "操作数": ["Weight"]
        },
        "匹配规则": "$GreaterThan$",
        "匹配值": "20"
      }
    }
  ]
}
```
**说明**：目的国为美国且重量大于20kg，两个条件必须同时满足。

**OR 运算符**
```json
{
  "逻辑运算符": "OR",
  "条件组合": [
    {
      "条件": {
        "计算因子": {
          "运算符": "NONE",
          "操作数": ["Length"]
        },
        "匹配规则": "$GreaterOrEqual$",
        "匹配值": "120"
      }
    },
    {
      "条件": {
        "计算因子": {
          "运算符": "ADD",
          "操作数": ["Length", "Width", "Height"]
        },
        "匹配规则": "$GreaterOrEqual$",
        "匹配值": "260"
      }
    }
  ]
}
```
**说明**：长度大于等于120或长宽高之和大于等于260，任意一个条件满足即可。

**NOT 运算符**
```json
{
  "逻辑运算符": "NOT",
  "条件组合": [
    {
      "条件": {
        "计算因子": {
          "运算符": "NONE",
          "操作数": ["CustomerLevel"]
        },
        "匹配规则": "$Equal$",
        "匹配值": "普通"
      }
    }
  ]
}
```
**说明**：客户等级不等于普通，当客户等级不是普通时满足条件。

#### 3.4 特性
- **组合能力**：可以组合多个条件，构建复杂的逻辑关系
- **嵌套支持**：支持嵌套使用，实现多层逻辑判断
- **优先级**：NOT > AND > OR

### 4. 操作符

#### 4.1 定义
操作符用于条件判断中的比较操作，确定计算因子值与匹配值之间的关系。

#### 4.2 类型

| 操作符 | 含义 | 适用类型 | 说明 |
|--------|------|----------|------|
| $Equal$ | 完全等于指定值 | 所有类型 | 字符串或数值必须完全匹配 |
| $NotEqual$ | 不等于指定值 | 所有类型 | 字符串或数值不相等即满足 |
| $GreaterThan$ | 大于指定值 | 数值类型 | 数值大于匹配值 |
| $LessThan$ | 小于指定值 | 数值类型 | 数值小于匹配值 |
| $GreaterOrEqual$ | 大于等于指定值 | 数值类型 | 数值大于或等于匹配值 |
| $LessOrEqual$ | 小于等于指定值 | 数值类型 | 数值小于或等于匹配值 |
| $AnyContain$ | 数组中任意一个元素包含指定值 | 数组类型 | 只要有一个元素包含匹配值即满足 |
| $AllContain$ | 数组中所有元素都包含指定值 | 数组类型 | 所有元素都必须包含匹配值 |

#### 4.3 示例

**$Equal$ 操作符**
```json
{
  "条件": {
    "计算因子": {
      "运算符": "NONE",
      "操作数": ["CustomerLevel"]
    },
    "匹配规则": "$Equal$",
    "匹配值": "VIP"
  }
}
```
**说明**：客户等级必须完全等于"VIP"，适用于精确匹配场景。

**$NotEqual$ 操作符**
```json
{
  "条件": {
    "计算因子": {
      "运算符": "NONE",
      "操作数": ["CustomerLevel"]
    },
    "匹配规则": "$NotEqual$",
    "匹配值": "普通"
  }
}
```
**说明**：客户等级不等于"普通"，适用于排除特定值的场景。

**$GreaterThan$ 操作符**
```json
{
  "条件": {
    "计算因子": {
      "运算符": "NONE",
      "操作数": ["FeeWeight"]
    },
    "匹配规则": "$GreaterThan$",
    "匹配值": "20"
  }
}
```
**说明**：计费重量大于20kg，适用于需要数值比较的场景。

**$LessThan$ 操作符**
```json
{
  "条件": {
    "计算因子": {
      "运算符": "NONE",
      "操作数": ["FeeWeight"]
    },
    "匹配规则": "$LessThan$",
    "匹配值": "10"
  }
}
```
**说明**：计费重量小于10kg，适用于需要数值比较的场景。

**$GreaterOrEqual$ 操作符**
```json
{
  "条件": {
    "计算因子": {
      "运算符": "NONE",
      "操作数": ["FeeWeight"]
    },
    "匹配规则": "$GreaterOrEqual$",
    "匹配值": "22"
  }
}
```
**说明**：计费重量大于等于22kg，适用于需要数值比较的场景。

**$LessOrEqual$ 操作符**
```json
{
  "条件": {
    "计算因子": {
      "运算符": "NONE",
      "操作数": ["FeeWeight"]
    },
    "匹配规则": "$LessOrEqual$",
    "匹配值": "50"
  }
}
```
**说明**：计费重量小于等于50kg，适用于需要数值比较的场景。

**$AnyContain$ 操作符**
```json
{
  "条件": {
    "计算因子": {
      "运算符": "NONE",
      "操作数": ["SkuDeclareNameCnArr"]
    },
    "匹配规则": "$AnyContain$",
    "匹配值": "空调"
  }
}
```
**说明**：商品名称数组中只要有一个名称包含"空调"，就满足条件，适用于数组元素部分匹配的场景。

**$AllContain$ 操作符**
```json
{
  "条件": {
    "计算因子": {
      "运算符": "NONE",
      "操作数": ["SkuDeclareNameCnArr"]
    },
    "匹配规则": "$AllContain$",
    "匹配值": "电器"
  }
}
```
**说明**：商品名称数组中所有名称都必须包含"电器"，才满足条件，适用于数组元素全部匹配的场景。
